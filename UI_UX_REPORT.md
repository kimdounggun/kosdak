# UI/UX 사용성 분석 보고서

사용자 관점에서 "불편하거나 보기 힘든 점"을 중점적으로 분석했습니다.

## 1. 인지 과부하 (Cognitive Load)
- **문제**: `SymbolDetailPage`에 너무 많은 정보가 한 번에 노출됩니다. 차트, AI 리포트, 신뢰도 패널, 히스토리 패널 등이 스크롤 없이 한 화면에 꽉 차 있어 시선이 분산됩니다.
- **위치**: `frontend/src/app/symbols/[id]/page.tsx`
- **개선**:
    - **탭(Tab) 구조 강화**: '종합', '차트', 'AI 분석', '재무' 탭으로 명확히 분리하여 한 번에 하나의 맥락만 보여주세요.
    - **점진적 공개**: AI 리포트의 상세 내용(근거, 대안 등)은 '더 보기' 버튼을 눌렀을 때 펼쳐지도록 변경하세요.

## 2. 시각적 계층 구조 (Visual Hierarchy)
- **문제**: `AiReportViewer`에서 모든 섹션(추세, 매매가, 요약)이 비슷한 비중으로 보입니다. 사용자가 가장 먼저 봐야 할 "결론(매수/매도)"과 "진입가"가 눈에 확 띄지 않습니다.
- **위치**: `frontend/src/components/Dashboard/AiReportViewer.tsx`
- **개선**:
    - **핵심 정보 최상단 배치**: '강력 매수'와 같은 결론을 가장 크고 굵게, 색상(초록/빨강)을 입혀 최상단 카드에 배치하세요.
    - **폰트 크기 차별화**: 부가 설명(RSI 70 이상 등)은 폰트 크기를 줄이고 색상을 흐리게 처리하여 중요 정보와 대비를 주세요.

## 3. 모바일 사용성 (Mobile Usability)
- **문제**: `SymbolCard`의 삭제 버튼(X)이 모바일에서 터치하기에 너무 작습니다(padding 0.5). 실수로 터치할 가능성이 높습니다.
- **위치**: `frontend/src/components/Dashboard/SymbolCard.tsx`
- **개선**:
    - **터치 영역 확대**: 버튼 크기를 최소 44x44px로 확보하거나, 스와이프 제스처로 삭제 기능을 숨기세요.
    - **편집 모드**: 대시보드 상단에 '편집' 버튼을 두고, 편집 모드에서만 삭제 버튼이 크게 나타나도록 변경하세요.

## 4. 데이터 갱신 피드백 (Feedback)
- **문제**: 대시보드에서 데이터가 언제 갱신되었는지 알기 어렵습니다. 사용자는 "이 가격이 지금 가격인가?"를 끊임없이 의심하게 됩니다.
- **위치**: `frontend/src/app/dashboard/page.tsx`
- **개선**:
    - **상대적 시간 표시**: 각 카드 우측 상단이나 하단에 "방금 전", "5분 전"과 같이 갱신 시점을 명시하세요.
    - **새로고침 애니메이션**: 데이터를 불러오는 동안 카드가 깜빡이거나 로딩 인디케이터가 명확히 보여야 합니다.

## 5. 색상 대비 및 가독성 (Contrast & Readability)
- **문제**: `PortfolioWidget` 등에서 네온 컬러(#00F0A8)와 짙은 배경(#15171A)의 대비는 좋지만, 작은 폰트(10px)의 회색 텍스트(#gray-500)는 가독성이 떨어집니다.
- **위치**: `frontend/src/components/Dashboard/PortfolioWidget.tsx`
- **개선**:
    - **폰트 크기 조정**: 최소 폰트 크기를 12px로 상향 조정하세요.
    - **명도 대비 확보**: 보조 텍스트의 색상을 좀 더 밝은 회색(#9CA3AF)으로 변경하여 가독성을 높이세요.
